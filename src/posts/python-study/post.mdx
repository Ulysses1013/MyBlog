### python 最初の学習方法

---

<br />

ここが最初のつまずきポイントです。

まず<span className="line line-yellow">本で学習を始めるのは、
個人的にオススメしません。</span>

本の情報は更新されないので、<br/>
**古い情報**になります。

そのため、**最新のもの**とはエラー内容や対処法が違うと言った問題に直面する<br/>
**可能性がある**のでオススメしません。

**学習の初期段階**で使うのはオススメしない、<br/>
と言うだけで、<span className="line line-yellow">ある程度学習が進んだ段階では全く問題ないです。</span>

<font color="red">初期段階での学習方法</font>

- Progate
- Paiza ラーニング
  <br />
  <br />

<font color="red">オススメ理由</font>

- 開発環境を構築する必要がない
- 無料
  <br />
  <br />

**初心者のつまずきポイント**である<br/>
**開発環境構築**が必要ない。また**無料**<br/>
なのが大きな点だと思います。

Progate・Paiza ラーニングで学習すれば、<br/>
無料枠だけでも**基礎はある程度できるようになります。**

### オススメのエディタ

---

<br />

**エディタ選びもつまずきポイントの１つ**です。<br/>
しかし、Python のエディタは以下の 2 つを選べば問題ないです。

- Visual Studio Code(VSCode)
- PyCharm
  <br />
  <br />

基本的に<span className="line line-yellow">どんな言語をやるにしても<br/>
エディタは VSCode を使った方が良いです。</span>

他にも色々なエディタを使ってきましたが、<br/>
結局 VSCode に落ち着きました。

**VSCode のメリット**の一部を紹介したいと思います。

<font color="red">VSCode のメリット</font>

- ほとんどすべての言語に対応している
- 自動補完機能
- 便利なプラグインが多数ある
- 無料
  <br />
  <br />

これだけの機能が備わっているのにも関わらず、<br/>
**無料**なので、使わない理由がない。

ただ Python に関しては PyCharm もオススメです。<br/>
**PyCharm は Python 専用のエディタです。**

<font color="red">PyCharm のメリット</font>

- コードをすぐに実行できる
- 自動補完機能
- Python 特化の機能が多数備わっている
  <br />
  <br />

<span className="line line-yellow">
  Python でコードを書くなら PyCharm が良いです。
</span>
<br />
<br />

しかし、**デメリット**として Python にしか対応していないので、<br/>
**汎用性を考えたら VSCode に軍配が上がります。**

### Web アプリルート

---

<br />

**python の主流な Web アプリ開発フレームワーク**は、<br/>
以下の 2 つです。

- Flask
- Django
  <br />
  <br />

**フレームワーク**とは、<br/>
アプリケーションを開発する時に使われる機能を<br/>
搭載したソフトウェアのこと。

その中でも**Flask**は、<br/>
**マイクロ Web アプリケーションフレームワーク**に当たります。

「マイクロ」と言う名前の通り、<br/>
必要最低限の機能しか搭載していないので、

<span className="line line-yellow">他のフレームワークより動作が軽い</span>です。

そのため、**シンプルな Web アプリを開発する**のに適しています。

Django（ジャンゴ）は、Instagram などのアプリでも使われている、<br/>
**フルスタック・フレームワーク**。

Django の**特徴**は Web アプリを開発する際に、<br/>

<span className="line line-yellow">
  便利な機能を他のフレームワークより多数搭載している
</span>
点です。
<br />
<br />

なので、**本格的な Web アプリを開発する**のに適しています。

|          | Flask | Django |
| :------- | :---: | -----: |
| 処理速度 |   ◎   |      ○ |
| 機能数   |   ○   |      ◎ |

<br />
<br />

表にするとこんな感じです。
