どうも**カニヤ**です。

git で直ぐに push したいのに**毎回パスワード**を聞かれて、<br/>
ちょっとイラときた事ありませんか？

わたしは**git**に**push**するたびに、<br/>
パスワードを**毎回聞かれる**のが面倒くさかったので,

どうやったら**聞かれず**に済むか色々調べました。

その中で**SSH 化の方法**を見つけたのですが<br/>
その方法は**少し複雑**。

そのため、わたしはより**簡単な方法**を探し、発見した。

それは<span className="line line-yellow">アクセストークンの発行</span>です。

なので、今回は**そのアクセストークンの発行方法**を解説したいと思います。

### Access tokens 発行方法

---

<br />

はじめに**github**にログインします。

次に**自分のアイコン**を押して、<br/>
**Settings → Developer settings**に移動。

そして,**Personal access tokens** →<br/>
**Generate new token**を押します。

押すと**パスワード**を聞かれるので入力。

**Edit personal access token**と言う画面になったら、まず**Note**にトークン名を入力します。

また、**トークンに与える権限**を選べるので<br/>
そこは好きなように選んでください。

自分は**以下の 3 つ**を選択しました。

<blockquote display="info">

- repo
- admin:public_key
- user

</blockquote>

そして、下にある<font color="green">緑色のボタン</font>を押せば完了です。

その後、**key**が表示されますが、

<font color="red">注意！</font>

この key は**一度しか見れない**ので、

<span className="line line-yellow">忘れずにメモしておきましょう。</span>
<br />
<br />

### アクセストークンの使い方

---

<br />

さっそく使ってみましょう。

push すると**ユーザー名とパスワード**が聞かれるので、ユーザー名を入力する。

そして、パスワード欄に先ほど**メモした key**を入力する。

そうすると**push**が完了します。

これで**次から push してもパスワードを聞かれること**はありません。

### まとめ

---

<br />

以上、**git でパスワードを毎回聞かれないようにする方法**でした。

その簡単な解決方法は**アクセストークンの発行**でした。

たったこれだけで、**問題が解決する**ので、<br/>
皆さんもぜひ、**この方法**を試してみてください。

くどいですが**大事なこと**なので、もう一度言います。
